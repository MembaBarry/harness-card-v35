# 07 — Go / No-Go Checklist

Use this immediately before making the repo public.

## Go if all are true

- [ ] Correct repo ZIP/folder is being used.
- [ ] `README.md` is clear and not overclaiming.
- [ ] `DISCLAIMER.md` exists.
- [ ] `PRIVACY.md` says no automatic data collection.
- [ ] `SECURITY.md` warns against committing private data.
- [ ] Feedback collection is voluntary and sanitized.
- [ ] GitHub issue template exists.
- [ ] Seed data has no private personal information.
- [ ] No API keys, tokens, private keys, or `.env` files are present.
- [ ] `index.html` opens and works locally.
- [ ] Public name is marked as working/under review.
- [ ] Evidence claims are marked as early, not proven.

## No-go if any are true

- [ ] You find private transcripts.
- [ ] You find API keys or tokens.
- [ ] You find personal legal/medical/financial details.
- [ ] The README claims proof or universal effectiveness.
- [ ] The repo asks users to submit raw private chats.
- [ ] The demo does not load.
- [ ] The issue template encourages unsafe sharing.

## Final public-release sentence

Use this as the public posture:

```text
Harness Card V3 is an early local-first prototype and working-name project. It is designed to help users test structured repair cards for AI conversations that drift, loop, overexplain, restart, or make unsupported claims. The project collects only voluntary, sanitized repair-outcome feedback and does not need or want personal conversations.
```
